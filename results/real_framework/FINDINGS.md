## What this answers (reviewer §5.4)

- **R1 backend/attack confound removed.** The SAME 96 attacks now run on all three
  backends. `backend_by_attack.csv` shows the effect is backend-specific and
  attack-specific, no longer entangled: on **vector**, every attack type A1–A8 has
  ASR 1.0; on **summary**, A1/A3/A6 collapse to 0.33 while A2/A4/A5/A7/A8 stay 1.0;
  on **kv**, most sit at 0.75 (A5 higher at 0.92). The earlier "summary vs vector"
  comparison was confounded (A4-only on summary); it is now clean.

- **R2 two real frameworks.** LangChain and LlamaIndex agree closely. Vector differs
  only marginally (effect 0.99 vs 0.98; retrieve 0.99 vs 1.00) from genuinely
  different ANN implementations; summary/kv coincide because they share the same
  LM Studio summariser and tag-KV logic.

- **R3/R4/R7 real write, embedding, summarisation, TTL.** Writes go through the real
  store; retrieval is real ANN over real embeddings; A4 compaction is a real LLM call.

- **New empirical finding — real summarisation launders and partially destroys.**
  Under summary/none, **lineage recovery drops to 0.25**: real summarisation severs
  the provenance edge back to the injected origin in 75% of fired runs. This is the
  §5.3/§5.8 laundering, previously modelled synthetically, now **measured** on real
  frameworks. Simultaneously summary/none ASR is only 0.75 and effect 0.50: real
  summarisation is an *unreliable, accidental* partial mitigation — it sometimes
  drops the split/time-delayed payload (A1/A3/A6) but faithfully carries the
  event-/reconstruction-triggered ones (A2/A4). It cannot be relied on as a defense.

- **Core defense result holds on real memory.** The 3-stage minimal defense drives
  **effect → 0 and ASR → 0 on every real backend and both frameworks**, and restores
  lineage recovery to 1.0. The write→survive→retrieve→effect funnel shows where it
  cuts: save-time filtering drops write/survive to 0.77, and retrieval-time
  revalidation collapses retrieve (vector 0.17, kv 0.06; summary stays 0.77 at
  retrieve but effect is still 0 because execution-time authority refuses dispatch).

## Internal consistency

- ASR ≥ funnel `effect` (vector none: 1.00 vs 0.99) — the ~1% gap is early on-topic
  firing before the probe, consistent with the ASR/funnel reconciliation in the paper.
- `survive == write` by construction (nested funnel).

## Phase C — real planner + real memory, end-to-end (36 runs)

`scripts/run_phase_c.py` (raw: `phase_c_runs.csv`, aggregate:
`phase_c_by_difficulty.csv`). Real LLM planner **Llama-3.1-8B** driving real
LangChain memory; difficulty swept explicitly (answers reviewer Q#5).

- **ASR = 1.00 at every difficulty (easy/medium/hard) with no defense; 0.00 under
  minimal defense at every difficulty.** The full stack (real planner + real
  memory) reproduces the effect and the defense holds.
- **The deterministic harness under-states the summary-backend risk.** On summary,
  a real LLM planner reconstructs the payload the summariser preserved and reaches
  A4 ASR 1.00, vs 0.75 for the rule-based planner in Phase B. A capable model makes
  the laundered-summary attack *worse*, not better.
- Covered end-to-end: A2 (event, vector), A3 (split, vector), A4 (summary, real
  summariser) across 4 workloads. 9.4 min, 183 LLM planner calls.

## Phase D — long-horizon dormancy + cross-tenant (`dormancy.csv`, `cross_user.csv`)

**Dormancy** (10/50/100 filler turns, deterministic planner, no defense):
- vector A2 (event) and summary A4: survive/retrieve/effect = 1 at **all** N,
  including N=100 — a topically-embedded payload survives arbitrary dormancy, and
  the summary payload persists through repeated real re-summarisation. Latency is
  not a mitigation.
- vector A3 (split): effect falls 1 → **0** at N≥50 (fragments persist but
  co-retrieval among ~100 distractors fails). Fragmentation trades stealth for
  long-horizon fragility.

**Cross-tenant** (attack authored in tenant A, probe from tenant B, vector store):
- shared store (no isolation): leak = **1** in all 4 workloads — poison crosses
  the user boundary.
- tenant-scoped retrieval: leak = **0** everywhere — namespace isolation stops
  cross-tenant transfer (whereas cross-session A6 succeeds because memory is
  session-global within one user).

## Phase E — cluster bootstrap + compute cost (`cluster_bootstrap.csv`, `cost.csv`)

**Scenario-level cluster bootstrap** (§5.5d; 5000 resamples of whole scenarios,
real-LLM no-defense ASR, 32 scenarios × 3 reps):
- Llama-3.1-8B: 0.948, 95% CI [0.906, 0.990]
- Mistral-7B / Mistral-Small / Gemma: 1.000, [1.000, 1.000]
Clustering respects the 3-rep correlation; conclusion unchanged.

**Compute cost** (§6.5, real backend):
- save-time content/provenance check: **3.3 µs/call** (pure Python, 10^4 avg);
  retrieval/exec checks same order → sub-ms per-turn defense overhead.
- baseline agent cost (paid with or without defense): ~150 ms/turn embedding
  retrieval; ~3.8 s per real summarisation.
- storage at probe: 6 states, 93 tokens. Defense adds **zero model calls**.
- only non-trivial cost is human: 4 confirmations (Table pareto), not compute.

## Scope note (honest limits)

- Phase C composes both real halves on a subset; scaling to the full matrix and
  more planner models remains future work.
- Dormancy/cross-tenant use the deterministic planner (memory-mechanics effects).
- End-to-end latency delta is dominated by embedding HTTP jitter; the defense's
  own cost is the microbenchmark above, and it adds no LLM calls.
- Dormancy is the matrix default (short); 10/50/100-turn long-horizon is Phase D.
- cross-user/tenant partitioning is Phase D.
- Per-stage latency/token/storage cost is Phase E.
- summary/kv results coincide across frameworks because the summariser and KV logic
  are shared; only the vector backend exercises framework-specific ANN retrieval.
